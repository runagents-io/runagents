# Platform API Reference

RunAgents keeps the full REST API reference on the main docs site.

Use these links when you need the canonical platform contract or the interactive API explorers:

- [OpenAPI contract](https://docs.runagents.io/api/openapi/)
- [Redoc reference](https://docs.runagents.io/api/redoc/)
- [Swagger UI](https://docs.runagents.io/api/swagger/)

The RTD site stays focused on the Python SDK and MCP experience. The main docs site remains the source for:

- the full REST API surface
- resource-by-resource API guides
- generated Redoc and Swagger views

## Developer workflow

The most common path is:

1. use RTD for SDK or MCP usage
2. jump to the platform API reference when you need endpoint-level detail
3. return to the SDK docs for Python examples and typed client usage
